=========  МОДЕЛЬ РАСПРОСТРАНЕНИЯ  ========

Файл, точка входа для интерпретатора Python, вспомогательные файлы с кодом.
Файлы данных.



=============  МОДЕЛЬ ДАННЫХ  =============

1. Получение данных от пользователя

Запрос любых данных повторяется до получения корректных данных.
    
    команды главного меню
        
                              начать новую партию : new    : n : начать   : н :
                    загрузить существующую партию : load   : l : загрузка : з :
                         отобразить раздел помощи : help   : h : помощь   : п :
              создать или переключиться на игрока : player : p : игрок    : и :
                   отобразить таблицу результатов : table  : t : таблица  : т :
                             изменить размер поля : dim    : d : размер   : р :
                                            выйти : quit   : q : выход    : в :
    
    ход игрока
        
        цифровые символы
        номер свободной клетки
    
    имя игрока
        
        ... (используемые символы?)
        уникальность



2. Структуры данных
    
    статистика игроков — база игроков
        
        {
          'игрок_1': {'побед': 2, 'поражений': 3, 'ничьих': 4},
          'игрок_2': {'побед': 0, 'поражений': 0, 'ничьих': 0},
        }
    
    сохранённые партии — база сохранений
        
        {
          ('игрок_1', 'игрок_2'): ([5, 3, 9], 3),
          ('игрок_2', 'игрок_1'): ([7, 9, 10, 22, 1, 20], 5),
        }
    
    сделанные ходы                                   1 | 2 | 3 
                                                    ———————————
        {5: 'X', 3: 'O', 9: 'X'}                     4 | 5 | 6 
                                                    ———————————
        [5, 3, 9]                                    7 | 8 | 9 
    
    выигрышные комбинации
        
        [
          {1, 2, 3},
          {4, 5, 6},
          {7, 8, 9},
          {1, 4, 6},
          {2, 5, 8},
          {3, 6, 9},
          {1, 5, 9},
          {3, 5, 7},
        ]


3. Файлы данных
    
    статистика игроков — players.ini
        
        [игрок_1]
        побед = 2
        поражений = 3
        ничьих = 4
        
        [игрок_2]
        побед = 0
        поражений = 0
        ничьих = 0
    
    сохранённые партии — saves.ttt
        
        игрок_1,игрок_2!5,3,9!3
        игрок_2,игрок_1!7,9,10,22,1,20!5



=====  ПОСЛЕДОВАТЕЛЬНОСТЬ ВЫПОЛНЕНИЯ  =====

1. Чтение данных из файлов

2. ЕСЛИ база игроков пустая:
        вывод полного раздела помощи

3. Запрос имени игрока

4. ЕСЛИ имя игрока отстутсвует в базе игроков:
        добавление игрока в базу игроков
        обновление файла статистики игроков

5. Авторизация игрока

6. Добавление имени игрока в список имён активных игроков
   Добавление объекта функции хода игрока в список функций активных игроков

# суперцикл
    
    6. Ожидание ввода команды игрока
    
    # new
        
        # настройка партии
        
        7. Запрос режима игры
           
           ЕСЛИ выбран режим ОДИН ЧЕЛОВЕК:
                запрос уровня сложности бота
                добавление имени бота в список имён активных игроков
                добавление объекта функции хода бота в список функций активных игроков
           
           ЕСЛИ выбран режим ДВА ЧЕЛОВЕКА:
                повтор шагов 3, 4 и 6
        
        # игровой процесс — цикл до максимально возможного количества ходов
            
            8. Запрос хода игрока
               
               ЕСЛИ пустая строка:
                    обновление базы сохранений
                    обновление файла сохранений
                    переход к шагу 6
            
            ИЛИ
            
            8. Вычисление хода бота
            
            9. Добавление хода в структуру сделанных ходов
            
            10. Вывод игрового поля со сделанным ходом
            
            11. ЕСЛИ достигнута победа:
                     переход к шагу 13
            
            12. Переход к шагу 8
        
        13. Обновление базы игроков
            Обновление файла статистики игроков
    
    # quit
        
        14. Выход из суперцикла

15. ...
